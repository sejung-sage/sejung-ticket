# PRD — 세정학원 강의실 가동률 대시보드

> 작성일: 2026-06-02 · 상태: **컨펌 대기 (Draft v1)**
> 데이터 소스: Supabase `public.aca_tickets` (419,035행) → `analytics` 스키마 정제 View (읽기 전용)

---

## 1. 배경 & 목표

세정학원은 `aca_tickets`에 "학생 1명 × 특정 날짜의 1수업"을 1행으로 적재한다(결제·미납·강의실·교사·강좌·정원·가격 포함). 이 원천 데이터를 조합해 **강의실이 시간적으로 얼마나 채워져 운영되는지(가동률)**, 그리고 **각 강좌가 정원 대비 얼마나 찼는지(충원율)** 를 한눈에 보는 운영 대시보드를 만든다.

### 목표
- 강의실(28개)·건물(5개 관)별 **시간 가동률**을 기간/요일/시간대로 시각화
- 비어 있는 시간대·저활용 강의실을 식별해 **공간 운영 최적화** 의사결정 지원
- 부가적으로 **충원율(정원 대비 수강)** 과 **미납률**을 함께 노출

### 비목표 (이번 범위 아님)
- 원천 데이터 수정/쓰기 (전부 읽기 전용)
- 강의실 예약/배정 기능 (분석·조회 전용)
- 학생 개인정보 상세 관리

## 2. 핵심 지표 정의

### 2-1. 세션(Session) — 집계 기본 단위
> `(classroom, class_date, class_name)` 조합 1개 = 강의실에서 실제 열린 1회 수업
- **수강 학생수** = 해당 세션의 행 수 (= 결제+미납 학생 합)
- **시간 구간** = `schedule_time` 파싱 → `[시작, 종료]` (분 단위)
- **수업 길이(duration)** = 종료 − 시작

### 2-2. 강의실 시간 가동률 (Primary) — "강의실 가동률"
```
일별 강의실 가동률 = (해당 강의실·날짜에 점유된 시간) / (운영 가능 시간)
  · 점유된 시간 = 그 날 그 강의실 세션들의 시간구간 합집합(겹침 병합)
  · 운영 가능 시간 = 운영시간 베이스라인 (기본 09:00–22:00 = 13h, 설정 가능)
```
- 건물/전체 가동률 = (Σ점유시간) / (Σ운영가능시간)
- 요일×시간대 단위로도 집계 → **히트맵**

### 2-3. 충원율 (Secondary) — 정원 대비 수강
```
세션 충원율 = 수강 학생수 / class_capacity
```
- ⚠️ `class_capacity`는 물리 강의실 정원이 아니라 **강좌 모집 정원**임을 UI에 명시
- 집계 시 가중평균: Σ학생수 / Σ정원

### 2-4. 미납률 (Tertiary)
```
미납률 = 미납(결제전) 학생수 / 전체 학생수
```
- `payment_state` 정규화: `결제완료`/깨진`°áÁ¦¿Ï·á` → **완납**, `결제전`/깨진`°áÁ¦Àü` → **미납**

## 3. 데이터 레이어 (analytics 스키마 · 읽기 전용)

원천 `aca_tickets`는 **무수정**. 가공은 전부 `analytics` 스키마 View로 격리.

### 3-1. `analytics.fn_parse_time_range(text) → (start_min, end_min)`
`schedule_time`의 다양한 표기를 분 단위로 정규화하는 SQL 함수.

| 입력 예시 | 해석 | start–end (분) |
|---|---|---|
| `18:00-22:00` | 24h 표기 | 1080–1320 |
| `pm6-9` | 오후 6–9시 | 1080–1260 |
| `P3:30-6` | P=오후, 3:30–6 | 930–1080 |
| `am10-1` | 오전10–오후1 | 600–780 |
| `09:00-4:30` | 09:00–16:30(오후보정) | 540–990 |
| `P5` | 5교시/단일 → 길이추정 or null | (정책 결정 필요) |
| `상시`/`00:00-24:00` | 종일 | 0–1440 |
| 파싱 불가 | — | null (집계 제외, 카운트만) |

### 3-2. `analytics.vw_sessions`
1행 = 세션. 컬럼: `building`(관), `classroom`, `class_date`, `dow`(요일), `class_name`, `teacher_name`, `subject`, `class_grade`, `class_type1`(학원), `start_min`, `end_min`, `duration_min`, `student_count`, `capacity`, `paid_count`, `unpaid_count`, `time_parse_ok`(bool).
- 필터: `classroom IS NOT NULL` AND `class_date <> '2050-01-01'`

### 3-3. `analytics.vw_room_daily`
1행 = `(classroom, class_date)`. 컬럼: `building`, `occupied_min`(겹침병합), `operating_min`, `utilization`(가동률), `session_count`, `student_sum`, `capacity_sum`, `fill_rate`, `unpaid_sum`.

### 3-4. `analytics.vw_room_heatmap`
1행 = `(classroom, dow, hour_slot)`. 시간대(1시간 버킷)별 평균 점유 비율 → 히트맵용.

> View는 마이그레이션 SQL 파일(`supabase/migrations/`)로 버전관리. 무거우면 일부를 Materialized View + `pg_cron`로 전환(추후).

## 4. 화면 / 기능

### 페이지 1 — 개요 대시보드 (`/`)
- **상단 필터 바**: 기간(프리셋: 최근7일/4주/3개월 + 커스텀), 건물(관), 강의실, 학원(class_type1), 요일
- **KPI 카드 5개**: 평균 가동률 · 총 가동시간(h) · 총 세션수 · 평균 충원율 · 미납률
- **가동률 추이 라인차트**: 일/주 단위 전체 가동률 트렌드
- **건물별 가동률 막대차트**: 관별 비교
- **데이터 품질 배너**: "강의실 미기록 세션 N건은 가동률 집계에서 제외됨" 안내

### 페이지 2 — 강의실 히트맵 (`/heatmap`)
- **요일 × 시간대 히트맵**: 행=강의실(또는 관), 열=시간대(09–22시), 셀 색농도=평균 점유율
- 셀 클릭/호버 → 해당 슬롯 세션 요약
- 건물 토글, 기간 필터

### 페이지 3 — 강의실 랭킹 테이블 (`/rooms`)
- 강의실별 정렬 가능 테이블: 가동률 · 가동시간 · 세션수 · 충원율 · 미납률 · 주이용교사/과목
- 가동률 낮은 순/높은 순 정렬로 저활용 공간 식별
- 행 클릭 → 페이지 4 드릴다운

### 페이지 4 — 강의실 상세 드릴다운 (`/rooms/[room]`)
- 선택 강의실의 일자별 타임라인(간트형): 날짜별 세션 블록(시간×강좌×교사)
- 세션 목록 테이블: 시간 · 강좌 · 교사 · 학생수/정원 · 충원율 · 미납수 · 결제상태 분포
- 빈 시간대 하이라이트

## 5. 기술 스택 & 아키텍처

- **Next.js 16 (App Router) + React 19 + TypeScript + Tailwind** (스캐폴딩 완료)
- **데이터 접근**: 서버 컴포넌트 + `analytics` View를 `supabase-js`로 SELECT (집계는 DB View가 수행 → 앱은 가벼움)
  - 419k행 직접 fetch 금지. 항상 집계 View 경유.
- **차트**: `recharts` (라인/막대) + 커스텀 히트맵(div grid)
- **UI 컴포넌트**: `shadcn/ui` (테이블·카드·필터·날짜선택)
- **키**: 서버에서 `SUPABASE_SECRET_KEY`로 집계 View 조회 (RLS 우회). 클라이언트엔 집계 결과만 전달.
- **캐싱**: Next.js `cacheLife`/`revalidate`로 집계 결과 캐시(예: 10–60분), 과도한 DB 부하 방지

```
브라우저 ──▶ Next.js 서버 컴포넌트 ──▶ analytics.vw_* (집계) ──▶ aca_tickets(원본 read-only)
                    │
                    └─ 집계 결과만 클라이언트로 (recharts/테이블 렌더)
```
